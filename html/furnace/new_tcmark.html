After changing r theta z of any tc or he on the oven, you should update all three shared memory databases using "read parameters from oven". This makes certain that the menus display the same information as the V-computers are using. It doesn't actually affect oven performance but life can become very confusing if you don't do this.

-----

You should also update the tc marks used by ovenc.cl to make the odisp screens.

# delete old TC lists delete tc.[lwbma] verify+

# generate new lists tcmark aspect=l > tc.l tcmark aspect=w > tc.w tcmark aspect=b > tc.b tcmark aspect=m > tc.m tcmark aspect=a > tc.a

# delete the old graphics metacode files delete [lwbma].mc verify+

# generate new metacode files to cover the 512x512 graphics window graph tc.l point+ box- mark=cross szm=0.01 wx1=1 wx2=64 wy1=1 wy2=64 >G l.mc graph tc.w point+ box- mark=cross szm=0.01 wx1=1 wx2=128 wy1=-47 wy2=80 >G w.mc graph tc.b point+ box- mark=cross szm=0.01 wx1=1 wx2=64 wy1=1 wy2=64 >G b.mc graph tc.m point+ box- mark=cross szm=0.01 wx1=1 wx2=64 wy1=1 wy2=64 >G m.mc graph tc.a point+ box- mark=cross szm=0.01 wx1=1 wx2=64 wy1=1 wy2=64 >G a.mc

The definitions of the various oven aspects are found in "mirror$src/odisp/tc_he_bounds.h".
